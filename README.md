# Система помощи принятия решений при выборе терапии АГ

Готовый статический сайт для GitHub Pages. В проекте две страницы:

- `index.html` — ввод данных пациента
- `result.html` — вывод рекомендаций по лечению

## Как запустить локально

Достаточно открыть `index.html` в браузере. Для корректной работы модулей JavaScript удобнее использовать любой простой локальный сервер, например:

```bash
python -m http.server 8000
```

После этого откройте `http://localhost:8000`.

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите все файлы из этого архива в корень репозитория.
3. В настройках репозитория откройте **Pages**.
4. В качестве источника выберите ветку `main` и папку `/root`.
5. Сохраните настройки — GitHub Pages опубликует сайт.

## Что реализовано

- автоматическое определение категории/степени АД по САД и ДАД;
- ручной или автоматический выбор стадии;
- ручной или автоматический выбор риска;
- два режима определения риска: SCORE2 и клинический расчет;
- блок дополнительной информации со скрывающимися подблоками;
- опросник старческой астении;
- калькулятор CHA2DS2-VASc;
- режимы `старт терапии` и `коррекция терапии`;
- вторая страница с клиническим сценарием, шагом терапии, примерами препаратов и дополнительными рекомендациями.

## Допущения, которые я зафиксировал в коде

Поскольку исходная логика очень большая и в нескольких местах не полностью формализована, в прототипе приняты такие технические решения:

1. Для GitHub Pages сайт сделан полностью статическим, без сервера и базы данных.
2. Передача данных между страницами идет через `localStorage`.
3. В режиме `коррекция терапии` добавлен чекбокс `Целевой уровень АД не достигнут`, потому что без этого невозможно надежно определить следующий шаг терапии.
4. Для SCORE2 реализовано определение риска по введенному числу и возрастным порогам, как было описано в постановке. Там, где в логике не хватало промежуточных уровней, оставлен ближайший формализованный уровень.
5. Примеры препаратов на второй странице берутся из встроенного справочника МНН и доз. Для каждого шага выбираются репрезентативные примеры, а не полный перебор всех возможных комбинаций.
6. Часть формулировок рекомендаций и предупреждений выводится в виде текстовых блоков, чтобы не перегружать интерфейс.

## Ограничение

Это демонстрационный прототип rule-based CDS-инструмента. Перед реальным использованием в клинической практике требуется обязательная валидация логики и контент-ревью профильным специалистом.
